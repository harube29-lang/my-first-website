== SNS 사이트 DB 구조 정리 ==

users 테이블

-indexid
-아이디
-비밀번호
-닉네임
-프로필 이미지 url(랜덤 이미지api 활용)



게시물 정보 (posts 테이블)

-indexid
-작성자

-게시물
-게시물 이미지 url
-좋아요 수 
- 작성시간
-위치정보
-해쉬 태그
-연결된 댓글 테에블 id




댓글 정보 코멘트 테이블
- -indexid
-작성자
-댓글내용
-작성시간
-연결된 게시물 id

* 참고 이미지 랜덤 url은 https://unsplash.com/ko 이미지의 api 사용해보고 오류 발생 시 다른 랜덤 이미지나 api나 url 방식으로 대체
 

게시물 작성 방식:
- 1단계: 내용(캡션) 입력
- 2단계: Unsplash API로 랜덤 이미지 여러 개 보여주기
- 3단계: 마음에 드는 이미지 선택 (새로고침으로 다른 랜덤 이미지들 볼 수 있음)
- 4단계: 선택한 이미지 링크를 posts 테이블의 image_url에 저장

댓글 정보 (comments 테이블)
- 게시물 댓글을 위한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가)
  * 댓글 내용: (댓글에 쓴 글)
  * 댓글 작성자: (누가 댓글을 썼는지 - users 테이블과 연결)
  * 댓글 달린 게시물: (어떤 게시물에 댓글을 달았는지 - posts 테이블과 연결)
  * 댓글 작성일: (언제 댓글을 썼는지)

SNS 테이블 연결 관계 (단순화 버전)
- 사용자 게시물: 한 사용자가 여러 게시물 작성 가능
  - 게시물 작성 시 Unsplash API로 랜덤 이미지 선택
  - 선택한 이미지 링크를 posts 테이블에 저장
- 게시물 내 좋아요: 각 게시물마다 likes_count 필드로 좋아요 수 관리 (별도 테이블 없음)
- 사용자 댓글: 한 사용자가 여러 게시물에 댓글 가능
- 게시물 댓글: 한 게시물에 여러 사용자가 댓글 가능
